# 回看方法

光记不看 = 存钱不查余额。回看才是觉察。

## 前置：数据提取

回看前先运行脚本获取结构化数据，避免逐文件读取浪费上下文：

```bash
python scripts/extract_tracking.py tracking/
```

输出为 JSON 数组，每条包含：`date`, `emotion_score`, `emotion_word`, `tags`。

---

## 周回看

**时间**：每周日晚 · 5 分钟

**分析步骤**：

1. 情绪评分走势（周一 → 周日），画 ASCII 折线
2. 高频情绪词 Top 3
3. 哪天最好？哪天最差？差的那天的触发事件是什么？
4. 是否出现与之前相似的重复模式？（如连续几周周三下午低分）→ 标记 `[pattern-candidate]`
5. 将结论注入 review-conductor 周复盘的「维度 3：待改进项」

**输出示例**：
```
本周 tracking 分析（6/23-6/29）：
情绪走势：6 → 5 → 7 → 8 → 4 → 6 → 7
高频情绪词：满足(3次)、焦虑(2次)、充实(2次)
最好：周四 评分8（完成了设计文档）
最差：周五 评分4（临时被打断3次）
模式候选：周五下午评分连续2周偏低 → 观察第3周
```

---

## 月回看

**时间**：每月最后一天 · 15 分钟

**分析步骤**：

1. 情绪低谷通常在什么场景？
2. 高能量时段有什么共同规律？（运动了？深度工作完成了？）
3. 本月 vs 上月情绪趋势
4. 某类触发出现 ≥3 次 → 创建/更新 `review/patterns/[模式名].md`

**输出模板**见 `assets/review-template.md`。

---

## 模式发现规则

| 条件 | 动作 |
|---|---|
| 同一情绪词 ≥4 次/月 | 在月度分析中标注，建议关注触发场景 |
| 同一触发场景 ≥3 次/月 | 写入 `review/patterns/[场景名].md` |
| 同一时段评分偏低 ≥3 周 | 标记为 `[pattern-candidate]`，下月继续观察 |
| 连续 2 月某模式持续 | 将 `[pattern-candidate]` 提升为确认模式 |

---

## 约束

- 数据不足不强行分析：<5 天 → "再记几天才能看出规律"
- 分析结果写入 review/，不写入 tracking/
- 不比较不同用户的评分，只做个体内趋势比较
